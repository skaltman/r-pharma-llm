Tools

So how do we solve these problems? Tools! Also called functions or function calling.

a.k.a. functions, tool calling or function calling

  • Bring real-time or up-to-date information to the model

  • Let the model interact with the world

Tools are extra capabilities that you give to the LLM, like the ability to search the R/Pharma 2025 schedule or check the weather. These capabilities are things that the model can‚Äôt do on its own. The goal is to let the model pull in real-time or up-to-date information as needed.

They can also give the model new abilities to take action on behalf of the user.

How does tool calling work?

Let me walk you through exactly how tool calling works step by step. This is the key pattern you need to understand.

The conversation starts with a user sending a message to the chatbot.

In this example, the user asks whether they can go to the pool today - which requires knowing the weather.

The model doesn‚Äôt have access to real-time weather data, just like we saw earlier with Minneapolis.

Now imagine we‚Äôve told the LLM that it has access to a tool called get_weather(). Knowing that it needs weather information and doesn‚Äôt have access to real-time data, the model decides to use the weather tool.

It sends back a message that includes a tool call - basically instructions to call the get_weather() function with the appropriate location argument. The model uses its training data to figure out the right location format (like a ZIP code).

The programmer takes that tool call message and executes the actual function - in this case, querying a weather API with the location the model specified.

The weather API sends back some data:

  • The forecast is mostly sunny
  • High of 98¬∫F
  • Low of 78¬∫F

The programmer sends that data back to the LLM. Note that it‚Äôs often in a pretty raw format, like JSON. Not the kind of thing you‚Äôd want to read directly unless you‚Äôre the kind of person who likes reading computer data formats.

The LLM doesn‚Äôt mind JSON though, so it reads the data and generates a response that incorporates the data from the tool call.

The LLM can‚Äôt run code by itself!

This is crucial to understand - the LLM doesn‚Äôt execute tools. It just asks you to run them.

The LLM also chooses the arguments for the tool. Here it picks the location based on the conversation.

It uses its knowledge to fill in the right values - using information from its training data.

The LLM can also call multiple tools in a single response if it needs to gather different information.

Tools in R

Step 1: Write a (normal) R function

get_weather <- function(zip_code) {
  # use weather API to get weather for that zip
  # ...
}

Step 2: Define the tool

Here‚Äôs the pattern for defining tools in ellmer. You provide the function, a description, and argument types.

tool_get_weather <- tool(
  tool_fn,
  description = "How and when to use the tool",
  arguments = list(
    .... = type_string(),
    .... = type_integer(),
    .... = type_enum(
      c("choice1", "choice2"),
      required = FALSE
    )
  )
)

Step 2: Define the tool

Here‚Äôs the pattern for defining tools in ellmer. You provide the function, a description, and argument types.

tool_get_weather <- tool(
  get_weather,
  description = 
    "Gets the current weather for a specified zip code",
  arguments = list(
    zip_code = type_string("The zip code.", required = TRUE)
  )
)

Step 3: Register the tool

chat$register_tool(tool_get_weather)
